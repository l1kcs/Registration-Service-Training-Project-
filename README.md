# 🚀 Registration Service (Training Project)

Цей проєкт є частиною навчального курсу з розробки на **Java** та присвячений відпрацюванню навичок написання бізнес-логіки валідації та комплексного **Unit-тестування**.

---

## 📋 Опис завдання
Мета проєкту — реалізувати сервіс реєстрації користувачів (`RegistrationService`), який забезпечує перевірку вхідних даних перед збереженням у сховище. Проєкт демонструє роботу з шарами логіки (**Service**) та доступу до даних (**DAO/Storage**).

### 🛠 Функціональні вимоги (Бізнес-логіка)
Метод `register(User user)` дозволяє реєстрацію лише за таких умов:

* **Unique Login:** Користувач із таким логіном ще не існує в `Storage`.
* **Login Length:** Логін містить не менше **6 символів**.
* **Password Length:** Пароль містить не менше **6 символів**.
* **Age Limit:** Користувач є повнолітнім (мінімум **18 років**).

> [!IMPORTANT]
> Якщо дані не валідні, система викидає кастомне **unchecked exception**.

---

## 🧪 Тестування
Основний акцент у проєкті зроблено на максимальному покритті коду тестами.

* **Конвенція іменування:** `<methodUnderTest>_<state>_<expectedBehavior>`
    * *Приклад:* `register_invalidAge_notOk`
* **Інструменти:** `JUnit`, `AssertJ`.
